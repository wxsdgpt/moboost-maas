# UI：Admin 模块

## 页面

- `/admin` — 总览
- `/admin/users` — 用户列表与详情
- `/admin/credits` — 积分账本
- `/admin/events` — 事件日志

## 信息架构

```
┌─────────┬──────────────────────────────┐
│ Admin   │  当前页面标题                │
│ Sidebar ├──────────────────────────────┤
│         │  AdminMutationBanner（写操作时） │
│  Users  ├──────────────────────────────┤
│  Credits│  数据表格（可搜索、分页）     │
│  Events │                              │
└─────────┴──────────────────────────────┘
```

## 视觉

- 整体色调比主端更克制，背景 `--bg`，强调用 `--info` 而非 `--brand`（防误操作）
- 危险操作按钮：`--danger` 背景 + 白字

## 关键交互

- 所有写操作必须通过 `AdminMutationBanner`：
  - 显示将要执行的内容
  - 要求二次确认（输入"confirm"或点击两次）
  - 操作完成后刷新数据 + 显示 toast

## 验收走查

- [ ] 所有写操作都有二次确认
