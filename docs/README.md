# Moboost MAAS 产品设计框架

> 按「产品需求 → UI 设计 → 研发开发 → 测试」四层递进组织，所有新功能按此顺序落地。
> 核心原则：**功能模块高内聚、模块之间低耦合**。

## 核心模块

本产品当前包含 5 个核心业务模块，每个模块在 01–04 层都有独立文档：

- **Brief** — 营销简报生成（意图识别 → 事实收集 → 报告）
- **Report** — 营销报告展示与演化（/report, /evolution）
- **Landing** — 落地页生成与预览（/landing, DevicePreviewModal）
- **Video** — VEO 3.1 视频生成与播放（/api/generate-video）
- **Admin** — 后台管理（用户/积分/配置）

每个模块在每层都遵循固定模板（见各层 `_template.md`）。

## 使用流程

新增/迭代一个功能时，按顺序走完下列检查项：

1. **PRD**：在 `01-prd/<module>.md` 补一个"迭代条目"——说清楚用户故事、验收标准、非目标。
2. **UI**：在 `02-ui/<module>-ui.md` 补交互说明 + 视觉规范（或引用设计稿链接）。
3. **Dev**：在 `03-dev/<module>-dev.md` 落「对外接口契约 + 目录内文件说明」。
4. **Test**：在 `04-test/<module>-test.md` 新增对应测试用例（单测 / 集成 / 视觉测试页）。

每个迭代在 PR 描述里勾选这四项是否都更新过。
