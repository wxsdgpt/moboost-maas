### 2.2 交互状态矩阵

| 元素 | 默认态 | Hover | Active/Focus | Disabled | Loading |
|------|--------|-------|-------------|----------|---------|
| 主按钮 | 品牌紫底+白字 | 亮度+10% | scale(0.98) | 50% 透明度 | Spinner+文字 |

### 2.3 动效规范

| 动效类型 | 时长 | 缓动函数 | 触发条件 |
|----------|------|----------|----------|
| 页面切换 | 200ms | ease-out | 路由变化 |
| 模态弹出 | 250ms | ease-out | 用户操作 |
| 列表项出现 | 150ms | ease-in-out | 数据加载完成 |
| Toast 通知 | 300ms in, 200ms out | ease | 系统反馈 |

### 2.4 Loading 策略

| 场景 | 策略 | 实现 |
|------|------|------|
| 首屏加载 | 骨架屏 | `<Skeleton />` 组件 |
| 数据请求 (<1s) | 不显示 loading | debounce 300ms |
| 数据请求 (1-5s) | 行内 spinner | 按钮/区域内 |
| 数据请求 (>5s) | 全局进度条 | 顶部进度条 |
| AI 生成 (长时间) | 流式输出 + 进度 | SSE/逐段渲染 |

## 三、异常态设计

### 3.1 异常态 Checklist

每个页面/功能必须覆盖以下异常态，未覆盖的标注理由：

| 异常态 | 是否涉及 | 设计方案 | 检查 |
|--------|----------|----------|------|
| **空状态** (无数据) | 是/否 | [插画+引导文案+CTA] | [ ] |
| **加载失败** (网络错误) | 是/否 | [重试按钮+错误说明] | [ ] |
| **权限不足** (未登录/无权) | 是/否 | [引导登录/申请权限] | [ ] |
| **输入校验失败** | 是/否 | [行内错误提示] | [ ] |
| **操作超时** | 是/否 | [超时提示+重试] | [ ] |
| **并发冲突** (多人编辑) | 是/否 | [提示+刷新/合并] | [ ] |
| **数据量极大** (分页/虚拟滚动) | 是/否 | [分页/无限滚动] | [ ] |
| **服务降级** (后端不可用) | 是/否 | [友好提示+降级方案] | [ ] |
| **浏览器兼容** (旧浏览器) | 是/否 | [优雅降级/提示升级] | [ ] |

### 3.2 错误信息规范

| 错误类型 | 语气 | 示例 |
|----------|------|------|
| 输入校验 | 具体、指导性 | "邮箱格式不正确，请检查是否包含 @" |
| 网络错误 | 简洁、可操作 | "网络连接失败，请检查网络后重试" |
| 服务错误 | 歉意、给方案 | "服务暂时不可用，请稍后再试" |
| 权限错误 | 说明原因 | "此操作需要管理员权限" |

## 四、分支逻辑设计

### 4.1 决策树

```
用户操作
├── 条件 A 满足
│   ├── 子条件 A1 → 结果 1
│   └── 子条件 A2 → 结果 2
├── 条件 B 满足
│   └── 结果 3
└── 条件都不满足
    └── 默认结果 (兜底)
```

### 4.2 权限矩阵

| 创建内容 | ✗ | ✓ | ✓ | ✓ |
| 编辑他人内容 | ✗ | ✗ | ✓ | ✓ |
| 删除内容 | ✗ | ✗ | ✓ | ✓ |
| 系统设置 | ✗ | ✗ | ✗ | ✓ |
