## 七、自动化测试规范

### 7.1 测试文件命名

```
# 前端 (Jest)
src/lib/[module]/__tests__/[function].test.ts
src/components/[name]/__tests__/[name].test.tsx

# 后端 (pytest)
tests/unit/test_[module].py
tests/integration/test_[endpoint].py
tests/e2e/test_[flow].py
```

### 7.2 测试数据管理

| 原则 | 做法 |
|------|------|
| 测试隔离 | 每个测试用例创建自己的数据，测试后清理 |
| 不依赖外部 | Mock 所有外部 API 调用 (OpenRouter, Supabase) |
| 可重复 | 使用 Faker 生成随机但确定性的测试数据 (固定 seed) |
| 快速执行 | 单元测试 < 10s，集成测试 < 60s，E2E < 5min |

### 7.3 CI 集成建议

```yaml
# .github/workflows/test.yml (示例)
test:
  steps:
    - run: npm test -- --coverage     # 前端单元测试
    - run: pytest --cov               # 后端单元测试
    - run: npx playwright test        # E2E 测试 (仅 main 分支)
```
