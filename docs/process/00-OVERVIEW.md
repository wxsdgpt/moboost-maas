# Moboost MAAS — Product R&D Process (L1)

> 产品研发全流程规范，确保每个功能从需求到上线都经过结构化的评审和验证。

## 流程总览

```
需求采集 → 需求评估 → 需求分析 → PRD → 设计评审 → 技术方案 → 研发 → 测试 → 发布 → 可观测
   ↑                                                                              |
   └──────────────── 反馈闭环：可观测数据驱动下一轮需求优化 ───────────────────────────┘
```

## 文档索引

| 阶段 | 文档 | 说明 |
|------|------|------|
| 需求 | `01-PRD-TEMPLATE.md` | PRD 文档模板，含需求采集表和 RICE 评估 |
| 设计 | `02-DESIGN-SPEC.md` | UI/UE 设计规范，异常态、分支逻辑 checklist |
| 研发 | `03-TECH-DESIGN.md` | 技术方案评审模板，接口契约，数据库设计 |
| 测试 | `04-TEST-SYSTEM.md` | 测试用例模板，多层测试 checklist |
| 可观测 | `05-OBSERVABILITY.md` | 日志规范，监控指标，Postmortem 模板 |

## 设计原则

**高内聚**：每个模块/组件只做一件事，做好一件事。
**低耦合**：模块之间通过明确的接口（API 契约）交互，不直接访问彼此内部实现。
**可观测**：每个关键节点都有结构化日志，出了问题能 5 分钟内定位到根因。
**可迭代**：流程本身也是产品——持续优化模板和 checklist。

## 使用方式

1. 新功能启动时，复制对应模板创建具体文档
2. 按阶段推进，每个阶段的 checklist 全部打勾后进入下一阶段
3. 发布后通过可观测数据评估效果，发现问题反馈到需求池
