## 完成的工作

### 1. 配置真实 OpenRouter API Key
- 在 `.env.local` 配置 `OPENROUTER_API_KEY`
- 确认 `OPENROUTER_BASE_URL=https://openrouter.ai/api/v1`

### 2. 修复图片生成流程
**问题：** `handleSubmit` 只触发了 ModelRouter 动画，没有调用真实 API。
**修复：** 在 `ModelRouter.onComplete` 回调里调用 `/api/generate`。

**问题：** 图片生成完成但页面不显示。
**根因：** Gemini 3 Pro 返回图片在 `message.images[]` 数组里，不在 `message.content`（为 null）。
**修复：** `src/app/api/generate/route.ts` 先解析 `message.images[]`。

### 3. 接通 VEO 3.1 视频生成
**问题：** `google/veo-3` 模型不存在（400 错误）。
**修复：** 改为 `google/veo-3.1`（全局替换 5+ 处）。

**VEO 3.1 异步 API 流程（通过 debug 端点探索发现）：**
```
POST /api/alpha/videos             → { id, status: "pending" }
GET  /api/alpha/videos/{id}        → { status: "pending"|"completed" }
GET  /api/alpha/videos/{id}/content?index=0  → 二进制视频数据
```
耗时 ~70s，约 $1.60/条。

### 4. 构建全局状态管理 Store
- 自定义 event-based store，基于 `useSyncExternalStore`
- 支持：projects, messages, assets, jobs, notifications
- `pollVideoJob()` 后台轮询（5s 间隔，最多 30 次）

### 5. 3 列工作区 `/project/[id]`
```
[80px 缩略图栏] | [flex 画布] | [380px 聊天+输入]
```
- 左：资产缩略图，悬停显示删除按钮，点击选中
- 中：全屏展示当前素材（图片/视频），工具栏（缩放/编辑/下载）
- 右：完整对话历史 + ModelRouter + ThinkingPanel + 底部输入框

### 6. 完善导航流程
- 首页提交 → 创建 project + job → 跳转 `/project/[id]` workspace
- workspace mount 后自动检测 `routing` job → 开始生成
- 项目列表页点击 → 跳转对应 workspace
- workspace 侧边栏自动隐藏（workspace 有自己的 3 列布局）

### 7. 修复 Notifications 组件
**问题：** `iconMap` 定义在 `Notifications` 函数内部，`NotificationBell` 组件无法访问。
**修复：** 移到模块顶层作为常量。

## 关键技术决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 状态管理 | 自定义 store + useSyncExternalStore | 无需重型库，React 18 原生 |
| 路由模式 | 首页输入 → 直接跳转 workspace | 所有对话在 project 内持久化 |
| 视频处理 | 异步提交 + 后台轮询 + Notification | VEO 3.1 需要 70s，不能阻塞 UI |
| Sidebar 隐藏 | 正则判断 pathname | workspace 自有 3 列布局，避免冲突 |
