# iGaming 广告素材本地化系统

## 项目定位

为 iGaming（体育博彩为主）公司的**内部营销团队**打造的**广告素材本地化工具**。

**核心：Localization, not Generation**
- 文字输入 → 文字输出（本地化，不创作）
- 图片输入 → 图片输出（保持构图，本地化元素）
- 视频输入 → 视频输出（保持画面/时长/比例/码率，本地化文字和音频）

系统不从 brief 生成新素材。AI 用于编辑和修改，不用于创作。

**输入技术属性 = 输出技术属性**。输入是 30 秒 16:9 MP4，每个输出也是 30 秒 16:9 MP4，只有和市场本地化相关的内容不同（语言、配音、合规元素）。

**不是本地化**：比例转换、多码率导出、时长剪辑——这些是渠道运营需求，由设计/运营工作流处理，不属于本系统。

## 用户角色

- **营销专员 / Ad Ops**：上传源素材、选择本地化策略、审阅合规报告、确认素材发布
- **品牌管理员（Brand Admin）**：管理品牌设置、品牌术语、品牌级合规规则 override、用户管理
- **系统管理员（System Admin）**：跨品牌只读可视、用户和品牌管理、成本报表、系统级监督

**重要说明**：本系统没有独立的"法务/合规官"角色。合规判断责任在营销/Ad Ops 团队，系统提供合规检查和完整留痕作为辅助工具。详见 `COMPLIANCE_GOVERNANCE.md`。

## 目标市场（V1）

美国 (US)、英国 (UK)、菲律宾 (PH)、印度 (IN)、巴西 (BR)、法国 (FR)、德国 (DE)、尼日利亚 (NG)

**市场监管复杂度排序**（从高到低）：DE > US > UK > FR > NG > BR > IN > PH

**子市场处理**：
- **US**：按州 sub-market 模型（`PER_STATE_OPERATING`）。全部 ~38 个合法州各自独立运营、独立规则包，V1 全覆盖。禁投州（CA、TX、UT、HI 等）在分发元数据强制排除。
- **NG**：按州 sub-market 模型（`PER_STATE_OPERATING`，与 US 同模型）。**2024 年 11 月最高法院裁决后**，博彩监管下放到州，NLRC 仅剩联邦首都区（FCT）管辖权。V1 优先覆盖 Lagos（LSLGA，最成熟）+ FCT（NLRC），其他州（Oyo、Rivers 等）数据模型预留，规则包按需求扩展。
- **IN**：按邦 blocklist 模型。一套印度素材，邦级禁投清单在分发时应用地理围栏。禁投邦包括 TN、AP、TS、OR、AS、NL 等，Karnataka 状态易变默认禁投。
- **UK**：两个子市场。默认 UK-GB（英格兰+苏格兰+威尔士，UKGC 统一监管）。可选 UK-NI（北爱尔兰，独立法律），品牌需显式启用。
- **BR**：V1 单一市场（SPA 联邦监管），数据模型预留子市场扩展。监控州级立法发展（Rio de Janeiro、São Paulo 已开始立法）。
- **DE、FR、PH**：联邦统一监管，单一市场处理（德国 GlüStV 2021 + GGL、法国 ANJ、菲律宾 PAGCOR）。

详见 `SUB_MARKETS.md`。

## 核心业务类型

产品广告 + 品牌广告，**重点是体育赛事博彩**。

## 核心用户体验

```
用户上传源素材（PSD 优先，扁平化图/视频兜底）
    ↓
系统解析为可本地化单元（Localizable Units）
    ↓
用户为每个单元选择本地化策略（有智能默认值）
    ↓
系统批量应用到 7 个市场（允许单市场覆盖）
    ↓
合规引擎检查（全部为警告，不阻止流程）
    ↓
用户审阅合规报告 + 确认素材（必经步骤，即使无违规）
    ↓
发布到投放渠道
```

## 关键产品决策

### 1. PSD-First, AI-Backup
- 源文件支持 PSD/AI 可编辑图层（优先）和扁平化图片
- 有图层 → 优先用确定性图层替换（快、准、可靠）
- 无图层 → 使用 Nano Banana 做精准编辑（AI 作为 fallback）

### 2. 每个单元用户可选策略
- 文字：保留 / 忠实翻译 / 轻度本地化 / 创译 / 用户自提供
- 视觉：保留 / 仅合规替换 / 文化本地化 / 自定义替换
- 音频：保留 / 仅加字幕 / 替换配音 / 原音+字幕

### 3. 批量默认 + 单市场覆盖
- 默认一次生成所有 7 个市场版本
- 允许为特定市场单独调整策略
- 合规规则按市场自动应用，不可覆盖

### 4. UI 简洁，后端透明
- 用户看到的只是最终成品
- 后端记录每个单元的策略、AI 调用、合规检查
- 审计时可完整复现

### 5. UI 全英文，内容保留目标语言
- 系统 UI 统一英文
- 处理的素材内容保留目标语言
- 法规引用保留原语言

## AI 能力清单

- **图像编辑**：Nano Banana (Gemini 2.5 Flash Image)
  - 图片内文字替换（保持字体、透视、光影）
  - 元素替换（人物、场景）
  - 元素删除（合规用）
- **视频编辑**：Veo 3.1
  - 视频内文字/字幕处理
  - 原生音频生成（配音替换）
  - 元素替换
- **文字处理**：LLM（Claude / GPT-4）
  - 忠实翻译
  - 轻度本地化
  - 创译（Transcreation）
- **图片理解**：多模态 LLM（取代 OCR）
  - 识别图片内文字和角色
  - 识别视觉元素
  - 初步合规筛查
- **合规检查**：多模态 LLM + 规则引擎 + 确定性叠加

## 关键设计原则

1. **保真度优先**：只改该改的，不碰不该碰的
2. **AI 做加速，不做决策**：合规最终判定权在法务
3. **确定性元素与 AI 生成分离**：logo、警语用代码层渲染
4. **按市场可配置**：规则、术语、策略支持市场级差异
5. **完整审计链路**：每次 AI 调用全量存档
6. **变更最小化验证**：未修改区域必须与源素材位对齐（perceptual hash）
